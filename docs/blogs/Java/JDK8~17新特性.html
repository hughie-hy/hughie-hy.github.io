    JDK8~17新特性

    hughie.cheng 5/14/2023 Java

    从JDK 8到JDK 17，Java平台引入了许多新特性和改进。以下是其中一些主要的新特性：

    # JDK 9:

    1. 模块系统（Java Platform Module System，JPMS）：引入了一种新的组织和部署Java代码的方式，以提高可靠性和安全性。
    2. JShell：一种交互式的Java编程环境，可以在其中直接编写和执行Java代码片段。
    3. 私有接口方法：接口可以包含私有的方法，这些方法可以被接口中的其他方法调用。

    # JDK 10:

    1. 局部变量类型推断：引入了var关键字，可以根据变量的初始化值进行类型推断，从而简化代码编写。
    2. 应用类数据共享（Application Class-Data Sharing）：通过在JVM启动时共享类元数据和字节码，提高了应用程序的启动时间和内存利用率。

    # JDK 11:

    1. HTTP客户端：引入了标准的HTTP客户端API，以便于进行HTTP通信。
    2. 单文件执行程序：可以直接将Java应用程序打包为可执行的单个文件。
    3. 长期支持（LTS）版本：JDK 11成为了Java平台的长期支持版本，获得了长期的技术支持和更新。

    # JDK 12:

    1. 微基准测试套件：引入了Microbenchmark套件，可以方便地进行Java应用程序的微基准测试。
    2. Shenandoah垃圾收集器（Shenandoah Garbage Collector）：一种低暂停时间的垃圾收集器，适用于大内存堆环境。

    # JDK 13:

    1. 文本块：引入了文本块语法，使得多行字符串的定义更加简洁易读。
    2. 动态CDS归档（Dynamic CDS Archives）：允许在运行时生成类数据共享（CDS）档案文件。

    # JDK 14:

    1. switch表达式增强：switch语句可以作为表达式使用，从而可以更简洁地编写代码。
    2. Records：引入了记录（Record）类，用于声明不可变的数据传输对象。

    # JDK 15:

    1. Sealed类和接口：引入了受限的类和接口，用于控制哪些类可以继承或实现它们。
    2. Pattern匹配：增强了instanceof运算符，使其可以直接获取匹配对象的成员。

    # JDK 16:

    1. 引入了一些改进的GC算法和垃圾收集器接口，如ZGC、Epsilon等。

    # JDK 17:

    1. Sealed类和接口的扩展：引入了可以在模块之间扩展受限类和接口的机制。
    2. 基于x86的64位平台上的向量API：提供了一组类和操作符，用于执行向量计算。
    3. 垃圾收集器接口增强：增加了新的垃圾收集器接口和功能。
