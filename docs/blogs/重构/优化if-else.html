    # 优化方案一：提前return，去除不必要的else

    如果if-else代码块包含return语句，可以考虑通过提前return，把多余else干掉，使代码更加优雅。

    优化前：

    if(condition){
    //doSomething
}else{
    return ;
}

    优化后：

    if（!condition）{
    return ;
}
//doSomething

    # 优化方案二：使用条件三目运算符

    使用条件三目运算符可以简化某些if-else,使代码更加简洁，更具有可读性。

    优化前：

    int  price ;
if(condition){
    price = 80;
}else{
    price = 100;
}

    # 优化方案五：使用 Optional

    有时候if-else比较多，是因为非空判断导致的，这时候你可以使用java8的Optional进行优化。

    优化前：

    String str = "hughie.cheng";
if (str != null) {
    System.out.println(str);
} else {
    System.out.println("Null");
}

    5
    6

    优化后：

    Optional<String> strOptional = Optional.of("hughie.cheng");
strOptional.ifPresentOrElse(System.out::println, () -> System.out.println("Null"));

    # 优化方案六：表驱动法

    表驱动法，又称之为表驱动、表驱动方法。表驱动方法是一种使你可以在表中查找信息，而不必用很多的逻辑语句（if或Case）来把它们找出来的方法。以下的demo，把map抽象成表，在map中查找信息，而省去不必要的逻辑语句。

    优化前：

    if (param.equals(value1)) {
    doAction1(someParams);
} else if (param.equals(value2)) {
    doAction2(someParams);
} else if (param.equals(value3)) {
    doAction3(someParams);
}
// ...
    1

    优化后：

    Map<?, Function<?> action> actionMappings = new HashMap<>(); // 这里泛型 ? 是为方便演示，实际可替换为你需要的类型

// 初始化
actionMappings.put(value1, (someParams) -> { doAction1(someParams)});
actionMappings.put(value2, (someParams) -> { doAction2(someParams)});
actionMappings.put(value3, (someParams) -> { doAction3(someParams)});

// 省略多余逻辑语句
actionMappings.get(param).apply(someParams);

    # 优化方案八：策略模式+工厂方法消除if else

    假设需求为，根据不同资产类型，查询对应类型的资产，优化前有以下代码：

        String assetType = "all";
    if ("all".equals(assetType)) {
        System.out.println("总资产");
     } else if ("direct".equals(assetType)) {
        System.out.println("直销资产");
    } else if ("proxy".equals(assetType)) {
        System.out.println("代销资产");
    }

    首先，我们把每个条件逻辑代码块，抽象成一个公共的接口，可以得出以下代码：

    // 资产查询接口
public interface IAssetService {
    void queryAsset();
}

    我们根据每个逻辑条件，定义相对应的策略实现类，可得以下代码：

    //全部资产策略实现类
public class AllAssetServiceImpl implements IAssetService {
    @Override
    public void queryAsset() {
        System.out.println("总资产");
    }
}
//直销资产策略实现类
public class DirectAssetServiceImpl implements IAssetService {
    @Override
    public void queryAsset() {
        System.out.println("直销资产");
    }
}
//代销资产策略实现类
public class ProxyAssetServiceImpl implements IAssetService {
    @Override
    public void queryAsset() {
        System.out.println("代销资产");
    }
}

    接下来，我们再定义策略工厂类，用来管理这些勋章实现策略类，如下：

    //勋章服务工产类
public class AssetServicesFactory {

    private static final Map<String, IMedalService> map = new HashMap<>();
    static {
        map.put("all", new AllAssetServiceImpl());
        map.put("direct", new DirectAssetServiceImpl());
        map.put("proxy", new ProxyAssetServiceImpl());
    }
    public static IAssetService getAssetService(String assetType) {
        return map.get(assetType);
    }
}
