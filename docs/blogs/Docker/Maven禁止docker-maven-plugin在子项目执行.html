    Maven禁止docker-maven-plugin在子项目执行

    hughie.cheng 6/18/2023 docker build

    # Maven禁止docker-maven-plugin在子项目执行

    在多模块工程下，在父工程下执行的插件默认会在其所有的子模块上也执行一遍，大多数情况下这是合理的，例如compiler这样的插件，但是有时候这种行为并不是开发人员所期望的，比如使用docker-maven-plugin构建推送镜像，当我们只希望在父工程下来执行一次该插件时，在多模块下，发现这个插件被执行了多次，导致出现多余镜像或报错，于是需要一种方法来禁用插件在子模块上的执行。

    父模块插件配置如下：

    <plugin>
	<groupId>com.spotify</groupId>
	<artifactId>docker-maven-plugin</artifactId>
	<version>0.4.13</version>
	<dependencies>
		<dependency>
			<groupId>javax.activation</groupId>
			<artifactId>activation</artifactId>
			<version>1.1.1</version>
		</dependency>
	</dependencies>
	<executions>
		<execution>
			<phase>deploy</phase>
			<goals>
				<goal>build</goal>
				<goal>tag</goal>
			</goals>
		</execution>
	</executions>
	<configuration>
		<maintainer>Rui fan &lt;hughie.cheng@hughie.com&gt;</maintainer>
		<imageName>nexus.hughie.com:7363/app5/hughie-ruoyi:latest</imageName>
		<baseImage>nexus.hughie.com:7363/openjdk-17-skywalking-agent:latest</baseImage>
		<registryUrl>https://nexus.hughie.com:7363</registryUrl>
		<serverId>ufos-docker-hub</serverId>
		<env>
			<TZ>'Asia/Shanghai'</TZ>
			<LC_ALL>en_US.UTF-8</LC_ALL>
		</env>
		<workdir>/app</workdir>
		<resources>
			<resource>
				<targetPath>/</targetPath>
				<directory>${project.basedir}/yudao-server/target</directory>
				<include>yudao-server.jar</include>
			</resource>
		</resources>
		<exposes>
			<expose>48080</expose>
		</exposes>
		<volumes>
			<volume>/tmp</volume>
		</volumes>
		<entryPoint>["/bin/sh", "-c", "set -e &amp;&amp; java -Xms1024m -Xmx1024m -jar /yudao-server.jar"]
		</entryPoint>
		<forceTags>true</forceTags>
		<pushImage>true</pushImage>
	</configuration>
</plugin>

    子模块插件配置

    <build>
    <plugins>
        <plugin>
            <groupId>com.spotify</groupId>
            <artifactId>docker-maven-plugin</artifactId>
            <version>0.4.13</version>
            <configuration>
                <skipDocker>true</skipDocker>
            </configuration>
        </plugin>
    </plugins>
</build>

    配置skipDocker依据

    <parameter>
  <name>skipDocker</name>
  <type>boolean</type>
  <required>false</required>
  <editable>true</editable>
  <description>Flag to skip docker goal, making goal a no-op. This can be useful when docker goal
	is bound to Maven phase, and you want to skip Docker command. Defaults to false.</description>
</parameter>
